- 还原点数据展示改为上下两张同步滚动的只读对比表，列宽按内容自适应，新增/删除/修改分别用绿色/红色/黄色背景标识。
- 批量替换改为双输入框：填写“要替换的内容”和“替换成”，例如 `_b` 替换为 `_c` 会把 `A_b` 处理为 `A_c`。
- 还原点的“保存当前文件”对比改为使用表格状态中的已保存快照，变量名等任意列的编辑都能在上下表中以黄色修改态显示。
- 变量表行数据接入本地持久化：保存时写入 `king-portable-toolkit.kio-rows.v1`，刷新后复制到文件夹的变量不会丢失。
- KIO 导出列补齐为 51 个完整字段，按真实 KIO 表头顺序导出，并补充常用默认值与中文枚举值转换。
- 工具包导航改为顶部图标菜单：只有当前选中工具显示文字；默认进入欢迎页，KIO 选中时才显示左侧项目树，Ping/端口为占位页。
- 表格“操作”列显示三点菜单和最近一次行操作图标按钮；三点左键菜单改为 fixed 定位并移除无意义的编辑/查看入口。
- KIO 导入按钮接入系统文件选择器，支持解析 CSV/TSV 为变量行；导入同名文件时在同目录自动追加序号生成新 CSV 节点。
- 修复珍珠山 GB18030 CSV 导入不显示：分隔符按逗号/制表符数量识别，避免表头缩进里的制表符误判为 TSV；导出 CSV 增加 UTF-8 BOM，避免常见表格软件打开中文乱码。
- 还原点面板移除演示假记录，改为读取真实保存快照；打开时只显示当前选中文件或文件夹命中的保存记录，未保存的字段清空会和最近保存版本做黄色差异对比。
- 变量数据类型和地址数据类型改为表格内下拉框：变量类型支持 IODisc/IOChar/IOByte/IOShort/IOWord/IOLong/IODWord/IOFloat/IOString/IOBlob/IODouble/IOInt64；地址类型支持 BIT/BYTE/SHORT/USHORT/LONG/LONGBCD/FLOAT/STRING/DOUBLE。
- 新增数据库直写测试脚本 `scripts/seed_all_type_test_csv.go`，已向默认 SQLite 写入 `全变量类型测试.csv`，包含 12 个变量数据类型测试行；前端选中 CSV 时会从后端 `GetKioProject` 加载数据库变量行用于表格和导出。
- 前端调试端口固定为 `34215`，Wails `frontend:dev:serverUrl`、Vite dev server 和 preview 均使用 `http://127.0.0.1:34215`。
- 工作区树新增刷新按钮；默认展开包含 CSV 的文件夹，便于直接看到数据库中新增的 `全变量类型测试.csv`。
- KIO 导出改为桌面端后端写盘：Wails 环境下 `SaveKioExportFiles` 先弹出目录选择器，再将 CSV 按 GB18030 编码写入用户选择的目录，避免 KIO 原软件导入 UTF-8 文件出现中文乱码；浏览器调试态仍保留 UTF-8 下载兜底。
- 按 KIO 原软件名称限制新增名称清洗与校验规则：变量名、通道名、设备名会移除/校验 `,;:+/*%&!.~|^<>=$[]{}:"'\?` 等无效字符，手工输入、批量操作、复制/新建和导出都会走同一规则；校验弹框会展示禁止字符和命中的行列。
- `scripts/seed_all_type_test_csv.go` 生成示例 CSV 时也会清洗变量名、通道名、设备名；已检查并重写当前数据库里的 `全变量类型测试.csv` 12 行名称字段。
- 同目录重复导出改为临时文件写入后系统替换，避免直接覆盖导致半写入或损坏；目标文件被占用时会失败并保留旧文件。
- 修复导入后 CSV 空白：导入流程会把前端解析出的表头、变量行和完整字段立即写入 SQLite，再从数据库加载当前 CSV；浏览器调试态也会立即写入本地行缓存和树缓存，避免新建节点后只有树记录没有变量数据。
- 修复校验范围错误：校验按钮只校验当前左侧选中层级下当前显示的变量，并在校验弹框显示范围，避免把其他 CSV 或历史缓存行的空字段混入结果。
- 根据 `读写权限.csv` 示例补充表格下拉枚举：读写权限支持 `只读/只写/读写`，并将启用、强制写入、滤波、冗余/转发开关等二选一字段接入 `是/否` 下拉。
- 修复还原点快照语义：保存当前文件时只保存当前选中层级的行作为当时快照；还原点不再用会随加载覆盖的 `savedRows` 冒充历史版本，避免对比内容看起来被当前数据污染。
- 还原点展示改为时间优先：新保存记录会记录本机 MAC 地址，列表和对比标题以保存时间为主体，副信息显示 MAC、当前范围和行数；旧记录不再展示“保存全部”作为主体。
- 根据 `测试测试.csv` 示例补充存储部分下拉枚举：历史记录方式支持 `每次采集记录/变化记录/不记录/定时记录`。
- 补齐存储和死区字段编辑：`DeadBand` 采集死区和 `HisDeadBand` 存储死区保留在全量模式中编辑；死区、周期、间隔等数值列统一使用数字输入。
- 优化列头右键编号操作：新增编号递减；编号递增/递减会从当前选中行或当前可见第一行识别末尾数字，不再要求用户理解 `{NN}` 模板，只填写起始编号和编号位数。
- 单元格右键菜单也补齐编号递增/递减入口，使用当前单元格内容推断编号模板，避免只能从列头三点菜单进入。
- KIO 表格容器新增专用宽滚动条样式，底部横向滚动条更容易拖拽操作。
