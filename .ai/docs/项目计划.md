# 三、关键原则

## 1. CSV 不是唯一数据源

导入 CSV 后：

```text
外部 CSV
  ↓
复制到软件工作区
  ↓
解析入 SQLite
  ↓
以后以 SQLite 为准
  ↓
导出时重新生成 CSV
```

这样用户即使移动或删除原始 CSV，软件内部数据也不会丢。

---

## 2. 项目下允许无限层级文件夹

不限制必须三层。

结构可以是：

```text
项目
└─ 文件夹
   └─ 文件夹
      └─ 文件夹
         └─ CSV
```

示例：

```text
大连某水厂
└─ 一期工程
   └─ 工艺段
      └─ 反应沉淀池
         └─ 阀岛
            └─ 东阀岛
               └─ 东阀岛2
                  ├─ 反应沉淀池A.csv
                  └─ 反应沉淀池B.csv
```

规则：

```text
项目是顶级单位
项目下可以无限新建文件夹
CSV 可以放在任意层级文件夹中
文件夹只负责分类
CSV 才是 KIO变量生成器处理对象
```

---

## 3. 所有字段都可以编辑

KIO CSV 有很多字段，不应该只支持常用字段。

设计原则：

```text
所有列都保存
所有列都可编辑
常用列默认显示
不常用列默认隐藏
用户可以展开高级字段
导出时所有字段都完整保留
```

也就是说：

```text
普通用户看到核心字段
高级用户可以展开全部 51 列
```

# 十九、最终一句话

最终设计可以这样描述：

> **便携工具包** 是一个基于 Wails、Go、React 和 SQLite 的本地桌面工具箱，正式版不启动 HTTP 服务、不占用端口。当前第一个工具是 **KIO变量生成器**，用于按项目和无限层级文件夹管理 KIO CSV，支持导入、编辑、新建、校验和导出。CSV 导入后写入 SQLite 数据中心，避免外部文件移动或删除导致数据丢失。KIO 变量的所有字段都可以编辑，常用字段默认显示，不常用字段默认隐藏，用户可展开高级字段或显示全部字段。表格支持丰富的右键批量操作，例如自动填充、批量替换、添加前缀、编号递增和 PLC 地址递增。顶级项目支持手动还原点，退出时会提醒用户是否创建还原点，方便版本管理和回滚。
